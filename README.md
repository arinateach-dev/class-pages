# 📚 class-pages

This repository hosts student-facing class pages as live web links. Each HTML file you upload becomes a page students can open in their browser — no app needed.

---

## 🔗 How the link is formed

Every file you upload gets a public URL that follows this pattern:

```
https://arinateach-dev.github.io/class-pages/YOUR-FILE-NAME.html
```

**Examples:**

| File you upload | Link students will use |
|---|---|
| `louboutin_a2_2.html` | `https://arinateach-dev.github.io/class-pages/louboutin_a2_2.html` |
| `french_b1_vocab.html` | `https://arinateach-dev.github.io/class-pages/french_b1_vocab.html` |
| `russian_intro.html` | `https://arinateach-dev.github.io/class-pages/russian_intro.html` |

> ⚠️ The link is case-sensitive. If your file is named `French_B1.html`, the link must have the exact same capitals.

## ➕ How to add a new page

### If the page has NO audio

1. Go to the repo on GitHub: [github.com/arinateach-dev/class-pages](https://github.com/arinateach-dev/class-pages)
2. Click the **Add file** button (top right of the file list) → **Upload files**
3. Drag your `.html` file into the upload area
4. Scroll down and click **Commit changes**
5. Wait about **1–2 minutes**, then your link is live ✅

---

### If the page HAS audio (mp3 files)

Your HTML file plays audio from a `.mp3` file. For this to work, **the mp3 must be uploaded to the same repository**.

1. Go to the repo on GitHub
2. Click **Add file** → **Upload files**
3. Drag **both** your `.html` file **and** your `.mp3` file into the upload area at the same time
4. Scroll down and click **Commit changes**
5. Wait about **1–2 minutes**, then test your link ✅

> 💡 Make sure the name of the mp3 in your HTML code matches the file you upload exactly, including uppercase/lowercase. For example, if your HTML says `audio.mp3`, the file you upload must also be called `audio.mp3` — not `Audio.mp3` or `AUDIO.MP3`.

---

## ✏️ How to update an existing page

If you need to replace a file with a newer version:

1. Go to the repo on GitHub
2. Click on the file you want to replace
3. Click the **pencil icon** (Edit) or the **upload icon** on the top right
4. Upload the new version with the **exact same filename**
5. Click **Commit changes**

GitHub will automatically replace the old version and the link stays the same.

---

## 💡 Tips

- **Keep filenames simple** — no spaces, no accents. Use underscores instead: `french_lesson_1.html` ✅ — `french lesson 1.html` ❌
- **The link only works once GitHub Pages has processed it** — this usually takes 1–2 minutes after uploading
- **You can upload multiple files at once** — just drag them all into the upload area together
